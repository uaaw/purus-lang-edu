# Purus Educationへのコントリビューション

[English version](CONTRIBUTING.md)

Purus Educationの改善に協力していただきありがとうございます。レッスン、ドキュメント、アクセシビリティ、デザイン、テスト、実装へのコントリビューションを歓迎します。

このプロジェクトへ参加する場合は、[行動規範](CODE_OF_CONDUCT-ja.md)に従ってください。

## 作業を始める前に

- 既存のIssueとPull Requestを検索し、作業の重複を避けてください。
- 大きな機能、設計変更、レッスンの再設計に着手する前にIssueを作成してください。
- 1つのコントリビューションは、1つの問題に集中させてください。
- 秘密情報、認証情報、個人情報、生成されたビルド出力をコミットしないでください。

## 開発環境

### 必要な環境

- Node.js 22以上
- npm

### インストール

```bash
git clone <your-fork-url>
cd purus-lang-edu
npm install
```

### 開発サーバーの起動

```bash
npm run dev
```

`http://localhost:3000`を開いてください。

削除または名前変更したルートが開発中に予期せず404になる場合は、サーバーを停止し、`.next/dev`だけを削除してから開発サーバーを再起動してください。

## 主な構成

- `src/app/`: Next.js App Routerのページ、レイアウト、APIルート
- `src/components/`: 共通UIコンポーネント
- `src/lib/lessons.ts`: 日英のレッスン内容と正規の採点条件
- `src/lib/grading.ts`: サーバー側のコンパイル、実行、採点
- `src/lib/i18n.ts`: 共通の日英UI文言
- `src/lib/share.ts`: X投稿文とURLの生成
- `tests/`: 採点と共有機能の回帰テスト

## 開発ガイドライン

### Next.js

このプロジェクトはNext.js 16を使用しており、以前のバージョンと異なる場合があります。Next.jsのコードを変更する前に、`node_modules/next/dist/docs/`内の関連ガイドを読み、現在のAPIと非推奨事項に従ってください。

### ユーザーインターフェース

- シンプルで分かりやすいUIを維持してください。
- キーボード操作、見やすいフォーカス表示、読みやすいコントラストを維持してください。
- ユーザー向け文言を変更する場合は、日本語と英語の両方を更新してください。
- 影響するページを`/ja`と`/en`の両方で確認してください。

### レッスンと採点

- 日本語と英語のレッスン内容を一致させてください。
- 正規の採点条件はサーバー側に定義し、クライアントから送信された期待値を信用しないでください。
- 正解、代表的な不正解、実行時エラー、境界値に対するテストを追加または更新してください。
- コンパイラ、実行処理、レッスン定義を変更した場合は、すべての公式解答が引き続き合格することを確認してください。

### 変更範囲とスタイル

- 小さくレビューしやすい変更を優先してください。
- 既存のTypeScriptとフォーマットのスタイルに従ってください。
- 同じPull Requestに無関係なリファクタリングを含めないでください。
- コードから明らかでない動作を説明する場合にのみコメントを追加してください。

## 検証

変更に関係するチェックを実行してください。

```bash
npm test
npx tsc --noEmit
npm run build
```

変更したすべてのソースファイルに対してESLintを実行してください。

```bash
npx eslint <changed-files>
```

個別のテストコマンドも利用できます。

```bash
npm run test:grading
npm run test:share
```

UIを変更した場合は、影響する日本語と英語のページを手動で確認してください。採点を変更した場合は、アプリケーション上で少なくとも1つの正解と1つの不正解を実行してください。

## Pull Request

1. 現在のデフォルトブランチから作業ブランチを作成します。
2. 目的を絞って変更し、内容が分かるコミットを作成します。
3. 動作を変更する場合は、テストとドキュメントを追加または更新します。
4. 関係する検証コマンドを実行します。
5. 問題、解決方法、検証方法を説明したPull Requestを作成します。

### Pull Requestチェックリスト

- [ ] 変更が記載された目的に限定されている。
- [ ] 日本語と英語の内容が一致している。
- [ ] 関係する自動テストが成功している。
- [ ] 型チェックが成功している。
- [ ] 変更したソースファイルがESLintを通過している。
- [ ] 実行時コードに影響する場合は、本番ビルドが成功している。
- [ ] 秘密情報、生成物、無関係な変更が含まれていない。
- [ ] 必要に応じてユーザー向けまたはコントリビューター向けのドキュメントを更新している。
